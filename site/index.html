A Claude Code plugin

Give Claude a better procedure, not a harder push.

Models don't get smarter when you ask them to think harder — they get better when you hand them a rigorous procedure for the situation in front of them. Better Thinking is a library of 130+ explicit thinking methods — decision analysis, root-cause investigation, competing-hypotheses evaluation — and a dispatcher that picks the right one automatically.

Not a persona

"Act like a senior analyst." "Think like a scientist." A costume changes tone, not rigor — the model still reaches for the first reasonable-sounding answer.

A checklist a skilled analyst would follow
  1. Form hypotheses
  2. Identify assumptions
  3. Design experiments
  4. Evaluate evidence quality
  5. Attempt falsification
  6. Update confidence
  7. Report remaining uncertainty
How it works

One command routes to the right method.

You don't need to memorize any skill names. Type /better-thinking at the start of any nontrivial task and the dispatcher does the rest.

Classify

Shape and stakes

Is this a decision, a diagnosis, a negotiation, research, a creative task? And how costly is it to get wrong or reverse?

Route

Match to method

The dispatcher reads a machine-indexed catalog of every skill and selects the ones suited to the task — not the ones easiest to reach for.

Calibrate

Depth follows stakes

A quick, low-stakes question gets a fast gut-check. A costly, hard-to-reverse decision gets the full structured treatment.

The catalog

130+ procedures, across 13 categories.

Atomic skills implement one cognitive move well — bayesian-updating, steelmanning, premortem. Composite skills orchestrate several atomics into a full workflow — decision-analysis, scientific-method, negotiation. Improve one atomic skill and every composite built on it improves too.

In practice

What routing actually looks like.

claude — better-thinking
/better-thinking should we migrate the billing service to a new database before Q3?
classifying task… decision, costly + hard to reverse routing → decision-analysis (composite) invoking decision-framing → options, decider, criteria, deadline invoking premortem → imagine the migration failed — why? invoking sensitivity-analysis → which assumptions the call hinges on — versus —
/better-thinking why did the retry logic double-charge three users last night?
classifying task… diagnosis, needs a fast, checkable answer routing → root-cause-investigation invoking five-whys → symptom to mechanism, stopping at a fixable cause
