# First Principles

Decompose a problem into base truths and rebuild the solution without inherited assumptions.

## Why

Most reasoning is reasoning-by-convention; when the convention is wrong or the situation is novel, it fails silently. This skill escapes inherited framing by rebuilding from what's actually true rather than what's customary.

## Use when / Don't use when

- **Use when:** the standard answer feels expensive, impossible, or arbitrary; entering a novel domain; someone says "that's just how it's done."
- **Don't use when:** the domain is mature and the convention is cheap to follow — most conventions encode real, hard-won learning.

## Inputs → Outputs

- **Inputs:** a problem plus the conventional approach or beliefs around it.
- **Outputs:** a list of verified base facts, a list of discarded assumptions, and a rebuilt solution path.

## Principles

- Distinguish physics from policy — what's truly impossible versus merely customary.
- Every "requirement" must trace to a base fact or be discarded.
- Rebuild bottom-up from surviving facts, not by patching the old design.

## Procedure

1. State the problem and the conventional solution.
2. List every assumption embedded in the conventional solution.
3. For each assumption, ask: is this a law of the domain, or an artifact of history, incentive, or habit? Keep only laws.
4. State the surviving base truths explicitly.
5. Rebuild a solution using only base truths, ignoring the old design.
6. Compare the rebuilt solution against the conventional one; explain each divergence.
7. Report which discarded assumptions carry the most risk if you're wrong about them, as residual uncertainty.

## Common mistakes

- Stopping decomposition one level too early, treating a convention as a law.
- Rebuilding into the same design anyway by anchoring on the familiar shape.
- Using it on well-solved problems where convention encodes real learning worth keeping.

## Examples

- Questioning why a rocket costs $60M when its raw materials cost $2M.
- Asking why a hiring process requires a specific degree rather than the capability it's meant to proxy for.
- Re-deriving a pricing model from unit costs instead of anchoring on competitor prices.
