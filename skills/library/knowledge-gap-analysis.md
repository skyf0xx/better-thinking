# Knowledge Gap Analysis

Map what's known, what's unknown-but-findable, and what's unknown-unknown for a task, so effort goes to the actual gap rather than re-covering the known.

## Why

Learning effort defaults to comfortable review of what's already known, because the genuinely unknown is harder to locate and harder to sit with. Explicitly mapping the boundary redirects effort to where it's needed.

## Use when / Don't use when

- **Use when:** starting to learn a new domain or skill for a specific task; feeling like "I don't know what I don't know."
- **Don't use when:** the domain is narrow and fully specified already.

## Inputs → Outputs

- **Inputs:** a task or domain to become competent in.
- **Outputs:** a three-part map — known, known-unknown, unknown-unknown, best-effort — with a prioritized closing plan.

## Principles

- Known-unknowns are addressable directly.
- Unknown-unknowns require deliberately probing the domain's edges — talking to experts, surveying the field's structure — since you can't look up a question you don't know to ask.
- Not all gaps matter equally — prioritize by what the task actually requires, not by completeness for its own sake.

## Procedure

1. State the task and what competent performance requires.
2. List what's already known and verified, via self-explanation if uncertain.
3. List known-unknowns: specific questions you know you need answered.
4. Probe for unknown-unknowns: what would an expert check that a novice wouldn't think to ask? Survey the field's outline or find a syllabus covering its scope.
5. Prioritize gaps by relevance to the task, not comprehensiveness.
6. Build a closing plan targeting the highest-priority gaps first.

## Common mistakes

- Only addressing known-unknowns while unknown-unknowns silently determine failure.
- Treating gap-mapping as a one-time exercise instead of revisiting as the task's real shape emerges.
- Over-indexing on comprehensive coverage instead of task-relevant priority.

## Examples

- Onboarding into a new codebase or domain.
- Preparing for a negotiation in an unfamiliar industry.
- A new manager mapping what they don't know about their team's actual work.
