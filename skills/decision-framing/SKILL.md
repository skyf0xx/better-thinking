# Decision Framing

Define what is actually being decided — options, decider, criteria, stakes, and deadline — before any evaluation begins.

## Why

Most decision failures are framing failures: a binary presented where the real option set is larger, criteria discovered mid-argument, nobody sure who actually decides. Ten minutes of framing prevents most of this.

## Use when / Don't use when

- **Use when:** any nontrivial choice, especially group ones; a debate is circling; someone asks "what do you think?" about an unframed decision.
- **Don't use when:** stakes are genuinely trivial — check reversibility first.

## Inputs → Outputs

- **Inputs:** a choice situation, however vaguely stated.
- **Outputs:** a decision statement: the question, options, decider, criteria, deadline, and the default if no decision is made.

## Principles

- The default (no-decision) path is always itself an option, and often the hidden favorite.
- Binary choices are usually a failure of option generation.
- "Who decides" and "who is consulted" must be distinct, named roles.
- A decision without a deadline is just a discussion.

## Procedure

1. Write the decision as a question with a deadline.
2. Name the decider, the consulted, and the merely informed, as three distinct groups.
3. State the default: what happens if no decision is made by the deadline?
4. List the options; if fewer than three exist, generate more.
5. Elicit criteria *before* comparing options, and note whose criteria they are.
6. State stakes and reversibility to size how much process this deserves.
7. Publish the frame and proceed — flag any remaining ambiguity in the frame as residual uncertainty.

## Common mistakes

- Evaluating options during framing, letting criteria get reverse-engineered to favor a favorite.
- Leaving the default path unexamined, when it's often quietly winning by inertia.
- Framing the question so only one option is a sane answer, without noticing.

## Examples

- "Should we rewrite the service?" reframed with the real option set: rewrite, strangle, patch, or freeze.
- A family relocation decision, framed with an explicit decider and deadline instead of drifting.
- A hiring committee discovering mid-debate that its members were silently optimizing for different criteria.
